Do you want to test on a desktop web device or on a mobile device? Expand the required section and click the preferred framework.

Test on desktop web devices

Selenium:  Requires a JDK, Selenium WebDriver, IDE, and Maven

Full prerequisites Java Development Kit 1.8 | Selenium WebDriver for Chrome (for running the local scripts) | An IDE of your choice, such as Eclipse or IntelliJ IDEA | If you work with Eclipse, TestNG for Eclipse and the Maven plugin | If you work with IntelliJ IDEA, the Maven plugin | Maven (download and install)
Test on mobile devices

  • Appium:  Requires a JDK, IDE, Maven, Android Studio, local Appium server

    Full prerequisitesJava Development Kit 1.8 | An IDE of your choice, such as Eclipse or IntelliJ IDEA | If you work with Eclipse, TestNG for Eclipse and the Maven plugin | If you work with IntelliJ IDEA, the Maven plugin | Maven (downloadandinstall) | Android Studio | Local Appium server, running at http://127.0.0.1/, port 4723For more information, see this introduction to Appium. For information on the Appium versions Perfecto supports, see the Selenium/Appium versionstab in the Supported platforms article.

  • Espresso:  Requires a Mac or Windows machine, Gradle, Java, Android Studio, Perfecto Gradle plugin

    Full prerequisites Mac or Windows machine | Gradle | Java | Android Studio with Android SDK

    In addition, you need access to the Perfecto Gradle plugin. You can download it either automatically, after adding the required lines of code to the build.gradle file, as described in step 2 below, or, if your organization does not permit direct download, by pre-downloading it to a local libs folder (see also Install the Perfecto Gradle plugin manually).

    The Perfecto Gradle plugin is not backward compatible. Make sure you always use + in the classpath to get the latest version of the build, as follows:

    com.perfectomobile.instrumentedtest.gradleplugin:plugin:+

  • XCUITtest:  Requires a Mac or Windows machine, Gradle, Apple Xcode

    In addition, you need access to the Perfecto Gradle plugin. You can download it either automatically, after adding the required lines of code to the build.gradle file, as described in step 2 below, or, if your organization does not permit direct download, by pre-downloading it to a local libs folder (see also Install the Perfecto Gradle plugin manually).
